# upsgi 0.1.0-rc1 release notes draft

## Summary
This draft release marks the first PSGI-first fork identity for upsgi.

## Highlights
- default build is PSGI-first
- default embedded logging bundle is:
  - `logfile`
  - `logsocket`
  - `rsyslog`
- `--static-map` remains baseline
- `--http-modifier*`, `--http-socket-modifier*`, and
  `--https-socket-modifier*` are accepted for migration compatibility but have
  no runtime effect
- `--perl-no-die-catch` is accepted for migration compatibility but has no
  runtime effect
- `--log-exceptions` is the canonical explicit PSGI debug flag
- fork regression and initial fault coverage now exist in `tests/fork/`

## Intentional behavior changes
- default PSGI exception catch/logging is off
- exception visibility is opt-in via `--log-exceptions`
- default docs and examples no longer center broader upstream multi-runtime
  breadth

## Migration notes
See `docs/upsgi/MIGRATION.md` for config mapping and safe migration steps.
