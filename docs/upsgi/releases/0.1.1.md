# upsgi 0.1.1 release notes

## Summary
upsgi 0.1.1 is the first post-maturation patch release of the PSGI-first fork.

## Highlights
- retained hardening from the Z cycle remains in place, including fail-closed handling for ambiguous duplicate `Host`, `Content-Length`, and `Transfer-Encoding` request-header cases
- low-risk request-time wins from Y7 improved the maintained local benchmark shapes without changing the supported fork contract
- maintained Y1 baseline and Y7 rerun artifacts are now part of the release record for this patch cycle
- the supported runtime story remains PSGI-first, HTTP-socket-first, logging-enabled by default, and explicit about compatibility-only inert legacy flags

## Measured local deltas carried into this release
Compared with the original maintained Y1 baseline, the Y7 rerun captured:
- dynamic GET logged: `+3.35%` req/s
- static-map GET logged: `+2.13%` req/s
- static-map HEAD logged: `+33.30%` req/s
- upload POST 4 KiB logged: `+30.92%` req/s
- upload POST 256 KiB logged: `+8.27%` req/s
- dynamic GET with logging disabled: `+22.98%` req/s

These numbers come from the maintained local fork harness and should be read as release-cycle checkpoint measurements, not universal deployment guarantees.
