# upsgi 0.1.2 release notes

Date: 2026-04-06

upsgi 0.1.2 is the public patch release that rolls up the post-0.1.1 hardening, PSGI strictness, HTTP/1.1 correctness, parser and PSGI bridge optimizations, lifecycle refinements, logging-path smoothing, and selective handler-inspired additions that were completed in the checkpoint series through Checkpoint 69.

## Highlights

### PSGI correctness and strictness
- strict duplicate request-header handling with an allowlist merge model
- stricter PSGI env, input, error, logger, and response semantics
- maintained coverage for delayed responders, streaming writers, cleanup handlers, harakiri, and informational responses

### HTTP and request-path hardening
- `Connection: close` now honored in the HTTP/1.1 path
- HEAD and chunked upload behavior covered in the maintained fork harness
- request-body spill threshold and read chunk sizing are now decoupled to reduce resident memory pressure

### Performance and operability
- parser micro-architecture now keeps most request header state in-place until final emission
- PSGI XS bridge does less copying on offset reads
- chain reload supports paced turnover with `--chain-reload-delay`
- master logging now uses bounded drain bursts with visible backpressure counters

### Selective handler-inspired additions
- added `psgix.informational` / `103 Early Hints` support for HTTP/1.1 PSGI requests

## Maintained fork test posture

The maintained `tests/fork/` harness now covers:
- startup and upload baseline behavior
- request and PSGI hardening paths
- PSGI env surface and body types
- buffered input semantics
- HTTP/1.1 keepalive, close, HEAD, and chunked uploads
- post-buffering memory behavior
- streaming and delayed responders
- cleanup and harakiri
- worker lifecycle controls
- accept-path distribution behavior
- logging backpressure
- informational responses

## Benchmark posture

The local benchmark review against the closest pre-optimization checkpoint showed modest wins on common GET and POST paths with no major common-path regression, plus more robust behavior under the worker-local mapped-socket accept shape.
