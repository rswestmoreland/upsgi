# upsgi 0.1.2 public release decision

Date: 2026-04-06

## Decision

Approve public repo upload and release preparation for upsgi 0.1.2.

## Why 0.1.2 is the right cut

This release bundles the main PSGI-first hardening and maturity work completed after 0.1.1:
- stricter PSGI boundary behavior
- stronger HTTP/1.1 correctness for practical PSGI hosting
- lower-risk parser and PSGI bridge optimizations
- worker lifecycle pacing and accept-path clarifications
- bounded logging-path smoothing and `psgix.informational` support

## Preconditions satisfied

- public source tree naming is centered on `upsgi`
- TOML-only Python build path is in place via `upsgiconfig.py`
- maintained fork regression and fault coverage exists under `tests/fork/`
- release/readiness review and local benchmark review have been completed
- packaging docs and release notes have been refreshed for 0.1.2

## Source artifact expectations

Generate and publish:
- `dist/upsgi-0.1.2-source.tar.gz`
- `dist/upsgi-0.1.2-SHA256SUMS.txt`

## Follow-up after upload

- run the maintained regression set in public CI
- build the default profile and verify `./upsgi --version` reports `0.1.2`
- cut the public tag and attach the source artifact/checksum pair
